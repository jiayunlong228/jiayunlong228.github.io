    数据处理流程与理论背景¶

    A/B 实验的数据分析可以分成三层：

    Text Only
    1
2
3
    数据口径：哪些用户、哪些天、哪些事件进入分析
统计方法：差异是否可能来自随机波动
业务判断：差异是否足够大，风险是否可接受

    这篇重点讲前两层。为了保持方法论清晰，不展开 SQL 实现。

    一、先定义数据口径¶

    1. 用户口径¶

    不同平台会给出不同层级的用户：

    用户层级 含义 用途
    符合目标人群 满足实验 Targeting 条件 判断实验覆盖范围
    被分流用户 被分配到 baseline 或 variant 分流校验
    激活用户 触发实验激活事件 结果统计口径
    实际曝光用户 真正看到实验功能或页面 诊断分析
    完成目标行为用户 产生转化、留存、收入等结果 指标分子

    Firebase 实验尤其要区分“被分流”和“被计入结果”。Activation event 会影响结果统计口径，不能混着看。

    2. 时间口径¶

    实验常见时间有三种：

    时间 含义
    分流时间 用户被分到某个实验组的时间
    激活时间 用户触发激活事件的时间
    行为时间 用户产生留存、收入、点击等行为的时间

    指标分析要统一以哪个时间为 D0。通常实验分析会以“进入实验或触发激活事件”为 D0，而不是用户安装日。

    3. 聚合口径¶

    实验分析应尽量按实验单位聚合。如果实验单位是用户，就先把指标聚合到用户级，再按组汇总。

    例如：

    • 是否 D1 留存：每个用户 0 或 1。
    • D7 LTV：每个用户进组后 7 日累计收入。
    • 人均广告展示：每个用户实验窗口内广告展示次数。
    • 关卡完成数：每个用户实验窗口内完成次数。

    不要把事件数直接当成独立样本。一个重度用户可能产生很多事件，但它们不是彼此独立的用户。

    二、指标体系¶

    1. 主指标¶

    主指标用于决定实验是否成功。它应该满足：

    • 和实验假设直接相关。
    • 能在合理周期内成熟。
    • 不容易被单个异常用户极端影响。
    • 实验前就确定，不在结果出来后更换。

    主指标越多，结论越难解释。一个实验最好只有一个主指标。

    2. 护栏指标¶

    护栏指标用于判断副作用。

    常见护栏方向：

    • 稳定性：崩溃、错误、卡顿。
    • 用户体验：卸载、投诉、差评、退出。
    • 长期价值：长期留存、LTV。
    • 生态平衡：某类核心行为是否被挤压。

    护栏指标不一定必须显著不变，但不能出现业务上无法接受的负向变化。

    3. 诊断指标¶

    诊断指标解释“为什么”。例如曝光、点击、漏斗分步、行为次数、分层指标等。

    诊断指标适合提出下一轮假设，不应轻易替代主指标成为最终决策依据。

    三、样本量估算¶

    样本量估算的目的，是在实验前判断实验能否回答问题。

    1. 核心概念¶

    概念 符号 含义 常用值
    显著性水平 \(\alpha\) 把无效方案误判为有效的概率上限 0.05
    检验功效 \(1-\beta\) 真实有效时被检测出来的概率 80% 或 90%
    基线值 \(p_0\) 或 \(\mu_0\) 对照组预期指标 用历史数据估计
    最小可检测效果 MDE 希望识别的最小差异 由业务判断确定

    MDE 要先由业务和产品判断给出，再反推样本量。不要反过来因为样本少，就把判断标准放宽。

    2. 比例类指标样本量¶

    适用于留存率、点击率、转化率、通过率等 0/1 指标。

    每组样本量近似为：

    \[ n \approx \frac{(Z_{\alpha/2}+Z_{\beta})^2[p_1(1-p_1)+p_2(1-p_2)]}{(p_2-p_1)^2} \]

    其中：

    • \(p_1\)：对照组预期比例。
    • \(p_2\)：实验组希望检测到的比例。
    • \(p_2-p_1\)：MDE 的绝对值。
    • \(Z_{\alpha/2}\)：显著性水平对应的分位数。
    • \(Z_{\beta}\)：功效对应的分位数。

    直觉：

    • 基线越接近 50%，方差越大，样本需求越高。
    • MDE 从 2pp 降到 1pp，样本量大约会变成 4 倍。
    • 如果希望更高功效，样本量也会增加。

    3. 均值类指标样本量¶

    适用于人均收入、人均时长、人均次数等指标。

    每组样本量近似为：

    \[ n \approx \frac{2(Z_{\alpha/2}+Z_{\beta})^2\sigma^2}{\delta^2} \]

    其中：

    • \(\sigma^2\)：用户级指标方差。
    • \(\delta\)：希望检测到的最小均值差异。

    均值类指标最关键的是方差。收入类指标通常长尾很强，方差大，需要的样本量会明显高于行为次数类指标。

    4. 实验周期估算¶

    先算进组天数：

    \[ D_{\mathrm{entry}} = \left\lceil \frac{n \times k}{d} \right\rceil \]

    其中：

    • \(D_{\mathrm{entry}}\)：进组天数。
    • \(n\)：每组所需样本量。
    • \(k\)：实验组数。
    • \(d\)：日均进入实验用户数。

    再加上指标成熟窗口：

    \[ T_{\mathrm{total}} = D_{\mathrm{entry}} + W \]

    其中，\(T_{\mathrm{total}}\) 表示总等待时间，\(W\) 表示指标成熟所需天数。

    如果主指标是 D7 留存，成熟天数至少是 7。如果还需要覆盖自然周期，通常还要保证实验至少跨过工作日和周末。

    四、分流效果验证¶

    分流验证发生在实验开始后、正式看结果前。

    1. SRM 检验¶

    SRM 是 Sample Ratio Mismatch，检查实际分组人数是否明显偏离预期。

    \[ \chi^2 = \sum_i \frac{(O_i-E_i)^2}{E_i} \]

    其中：

    • \(O_i\)：第 \(i\) 组实际人数。
    • \(E_i\)：第 \(i\) 组期望人数。

    两组 50/50 实验时，自由度为 1：

    结果 解释
    未发现 SRM 分组人数没有明显异常
    轻微 SRM 需要排查，但不一定直接废弃
    严重 SRM 实验随机性可能被破坏，不建议直接下结论

    常见 SRM 原因：

    • Targeting 配置错误。
    • 客户端版本逻辑不一致。
    • 参数拉取或激活时机不同。
    • 某一组埋点丢失。
    • 用户重复分组或跨组。

    2. 属性均衡检验¶

    SRM 只看人数，不看用户质量。属性均衡用于检查各组用户是否可比。

    建议检查：

    • 平台。
    • 国家。
    • 版本。
    • 渠道。
    • 新老用户比例。
    • 实验前历史行为。

    如果重要属性不均衡，结果可能需要分层分析，严重时应重做实验。

    如果某个环节在实验组和对照组之间差异异常，要先解释这个差异，再看主指标。

    五、显著性检验¶

    1. 比例类指标：两比例 z 检验¶

    两组比例：

    \[ \hat p_A = \frac{x_A}{n_A}, \quad \hat p_B = \frac{x_B}{n_B} \]

    合并比例：

    \[ \hat p = \frac{x_A+x_B}{n_A+n_B} \]

    标准误：

    \[ SE = \sqrt{\hat p(1-\hat p)\left(\frac{1}{n_A}+\frac{1}{n_B}\right)} \]

    检验统计量：

    \[ z = \frac{\hat p_B-\hat p_A}{SE} \]

    当样本量足够大时，可以用正态分布近似计算 p-value。

    2. 均值类指标：Welch t 检验¶

    两组均值差：

    \[ \Delta = \bar x_B-\bar x_A \]

    标准误：

    \[ SE = \sqrt{\frac{s_A^2}{n_A}+\frac{s_B^2}{n_B}} \]

    检验统计量：

    \[ t = \frac{\Delta}{SE} \]

    Welch t 检验不要求两组方差相等，通常比假设方差相等的 t 检验更稳健。

    3. 置信区间¶

    95% 置信区间可以写成：

    \[ 95\%\,CI = \Delta \pm 1.96 \times SE \]

    置信区间比单个 p-value 更适合产品决策，因为它同时表达了方向、大小和不确定性。

    六、多重比较和中途查看¶

    1. 多重比较¶

    如果同时看很多指标、很多人群、很多 variant，就会增加误判概率。常见处理方式：

    • 实验前锁定一个主指标。
    • 护栏指标用于风险判断，不随意找成功指标。
    • 分人群结果更多作为新假设。
    • 必要时做 Bonferroni 或 FDR 校正。

    2. 中途查看¶

    实验运行中可以监控质量和风险，但不应看到显著就停止。频繁查看并在结果好时停，会抬高假阳性概率。

    更稳妥的做法：

    • 实验前定义最短运行时间。
    • 实验前定义样本量目标。
    • 实验前定义严重风险的停止规则。
    • 正式结论等主指标成熟后再写。
