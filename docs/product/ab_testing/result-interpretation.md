# 如何解读实验结果

A/B 实验结果不应该被简化成“显著就上线，不显著就放弃”。正确的解读要同时看五件事：

1. 数据是否可信。
2. 效果方向是否符合假设。
3. 效果大小是否达到 MDE。
4. 置信区间是否可接受。
5. 护栏指标是否稳定。

## 一、先判断实验是否可信

正式解释结果前，先回答：

- 分流比例是否符合预期。
- 关键用户属性是否均衡。
- 实验组和对照组是否都正确触发埋点。
- 用户是否稳定留在同一组。
- 主指标是否已经成熟。
- 实验期间是否有版本发布、活动、渠道变化等外部干扰。

如果这些问题没有通过，后面的 p-value 和置信区间都要谨慎使用。

## 二、先限定因果结论边界

随机 A/B 实验最重要的价值，是在分流合理、口径一致、外部干扰可控时，支持“这次改动对这批实验用户造成了什么影响”的因果判断。

但它不能自动回答所有问题：

- 不能证明这个效果一定适用于所有用户、所有国家、所有版本。
- 不能直接解释深层原因，只能证明改动和指标变化之间存在实验条件下的因果影响。
- 不能排除所有异质性；整体正向可能来自某些人群强正向、另一些人群负向。
- 如果分流、激活口径或埋点被破坏，实验会退化成相关性观察。

所以结论建议写成：

> 在本次实验覆盖的人群、版本和统计窗口内，实验组相对对照组对 `[指标]` 产生了 `[方向和大小]` 的影响。

而不是直接写成：

> 这个功能一定能提升全量用户指标。

## 三、看点估计

点估计是当前样本观察到的差异。

比例类指标：

| 项目 | 数值 |
| --- | --- |
| baseline | 40.0% |
| variant | 42.0% |
| 绝对提升 | +2.0pp |
| 相对提升 | +5.0% |

均值类指标：

| 项目 | 数值 |
| --- | --- |
| baseline | 0.120 |
| variant | 0.126 |
| 绝对提升 | +0.006 |
| 相对提升 | +5.0% |

点估计回答的是“眼前看到的差异是多少”，但它不回答“这个差异有多稳定”。

## 四、看置信区间

置信区间表达不确定性。相比 p-value，它更适合用来做产品判断。

| 置信区间形态 | 解读 | 决策倾向 |
| --- | --- | --- |
| 整体大于 0，且下界达到业务要求 | 正向证据较强 | 可以考虑上线或放量 |
| 整体大于 0，但下界很小 | 统计上正向，业务价值不稳 | 看成本和护栏指标 |
| 跨过 0 | 当前证据不足 | 延长、重做或暂不采纳 |
| 整体小于 0 | 负向证据较强 | 停止或回滚 |

例如：

| 项目 | 数值 |
| --- | --- |
| 点估计 | +0.6pp |
| 95% CI | [+0.1pp, +1.1pp] |

说明当前估计是正向的，且置信区间没有跨过 0。但如果事前定义的 MDE 是 +0.5pp，区间下界 +0.1pp 就提醒我们：真实收益可能低于业务期望。

## 五、看 p-value

p-value 回答的是：

> 如果实验组和对照组真实没有差异，那么观察到当前这么大、甚至更大的差异，有多大概率只是随机发生的？

常见阈值：

| p-value | 常见说法 | 解读 |
| --- | --- | --- |
| < 0.001 | 高度显著 | 随机波动解释当前差异的可能性很低 |
| < 0.01 | 非常显著 | 证据较强 |
| < 0.05 | 显著 | 达到常用阈值 |
| >= 0.05 | 不显著 | 当前证据不足 |

注意：

- p-value 不是“实验成功概率”。
- p-value 不代表上线后一定有效。
- 样本很大时，微小差异也可能显著。
- 样本很小时，有价值的差异也可能不显著。

所以 p-value 要和效果大小、置信区间、MDE 一起看。

## 六、看是否达到 MDE

MDE 是实验前定义的“值得检测的最小效果”。解读时建议把结果分成四类：

| 统计结果 | 是否达到 MDE | 解读 |
| --- | --- | --- |
| 显著 | 达到 | 证据较强，进入上线评估 |
| 显著 | 未达到 | 统计显著但业务价值不足 |
| 不显著 | 点估计达到 | 方向可能对，但样本或波动不足以确认 |
| 不显著 | 未达到 | 暂无足够证据支持方案 |

这样可以避免把“显著性”当成唯一标准。

## 七、看护栏指标

主指标正向，不代表一定上线。护栏指标用于回答“这个收益有没有代价”。

护栏指标出现负向时，要判断：

- 负向变化是否统计显著。
- 负向变化是否达到业务风险阈值。
- 负向影响是否可逆。
- 是否只发生在某些人群或版本。
- 能否通过分层上线或参数调整降低风险。

如果主指标小幅提升，但护栏指标明显变差，通常不建议全量上线。

## 八、用诊断指标解释变化

诊断指标的价值，是帮助解释主指标为什么变化，而不是在主指标失败后临时寻找“赢了的指标”。

对收入或价值类指标，可以先用一个简化关系帮助拆解：

$$
\mathrm{LTV} \approx \mathrm{Retention} \times \mathrm{ARPDAU}
$$

这不是严格恒等式，但能帮助判断变化来源：

| 现象 | 可能解释 | 解读重点 |
| --- | --- | --- |
| 留存上升，ARPDAU 基本不变 | 更多用户留下来，单个活跃用户价值稳定 | 偏留存改善 |
| 留存基本不变，ARPDAU 上升 | 活跃用户的变现或行为强度提高 | 偏效率改善 |
| 留存下降，ARPDAU 上升 | 可能只留下了更高价值用户 | 警惕选择偏差和长期伤害 |
| 留存上升，ARPDAU 下降 | 留下了更多低价值或轻度用户 | 看 LTV 总量和护栏 |

如果主指标是 LTV，最终仍应以 cohort 口径的 LTV 作为决策指标。ARPDAU、eCPM、活跃用户人均时长等随机分母指标，更适合作为解释和定位问题的线索。

## 九、常见结果类型

### 1. 显著正向，护栏稳定

这是最理想的结果。

结论可以写：

> 实验组主指标显著高于对照组，提升幅度达到事前定义的 MDE。主要护栏指标未出现明显负向变化。建议进入放量或上线阶段，并继续观察长期指标。

### 2. 显著正向，但护栏变差

这类结果要谨慎。

结论可以写：

> 实验组主指标显著提升，但护栏指标出现负向变化。当前方案证明了作用方向，但全量上线风险较高。建议缩小适用人群、调整参数，或设计下一轮实验验证更温和方案。

### 3. 不显著，但点估计正向

不要直接说“有效”。

结论可以写：

> 实验组点估计为正，但置信区间跨过 0，当前数据不能排除随机波动。如果点估计接近或超过 MDE，可以评估是否延长实验。如果样本量已经足够，则暂不支持上线。

### 4. 不显著，且点估计接近 0

通常说明方案没有体现出可观察效果。

结论可以写：

> 实验组和对照组差异很小，当前数据不支持原假设。建议不继续推进该方案，优先复盘假设链路和诊断指标。

### 5. 显著负向

要及时止损。

结论可以写：

> 实验组主指标显著低于对照组，且不符合上线要求。建议停止实验或回滚方案，并复盘具体负向来源。

## 十、分人群结果怎么用

分人群分析很有价值，但也很容易产生误判。

建议原则：

- 实验前预设的人群分层，可以作为辅助决策。
- 实验后临时切出来的显著差异，更适合作为新假设。
- 分层越多，多重比较风险越高。
- 分层样本太小时，不要过度解释。

如果发现某个人群明显受益，可以设计下一轮分层实验，而不是直接把探索性结果当成确定结论。

## 十一、推荐结论模板

| 模块 | 写法 |
| --- | --- |
| 实验结论 | 本次实验主指标为 `[主指标]`，事前定义 MDE 为 `[阈值]`。实验组点估计为 `[数值]`，相对对照组提升 `[绝对提升 / 相对提升]`。统计检验结果为 `p-value = [数值]`，`95% CI = [区间]`。 |
| 数据质量 | 分流比例 `[正常 / 异常]`，关键属性 `[均衡 / 存在偏差]`。激活口径 `[清晰 / 存在问题]`，主指标成熟窗口 `[已满足 / 未满足]`。 |
| 风险判断 | 护栏指标 `[稳定 / 出现负向变化]`。负向变化主要体现在 `[指标]`，风险等级为 `[低 / 中 / 高]`。 |
| 决策建议 | 建议 `[上线 / 放量 / 延长实验 / 重做实验 / 放弃]`。原因是 `[是否达到 MDE、CI 是否可接受、护栏是否稳定]`。 |
| 后续动作 | 继续观察 `[长期指标]`，并基于 `[诊断发现]` 设计下一轮实验。 |

## 十二、最终判断顺序

可以用下面这个顺序做最后检查：

1. 分流和数据质量是否通过
2. 激活口径和成熟窗口是否正确
3. 因果结论是否限定在实验覆盖范围内
4. 主指标方向是否符合假设
5. 点估计是否达到 MDE
6. 置信区间下界是否可接受
7. p-value 是否支持统计显著
8. 护栏指标是否稳定
9. 分层和 cohort 是否一致
10. 上线收益是否大于成本和风险

实验分析的价值不只是判断“哪个版本赢了”，更是把不确定的产品判断拆成可验证、可复盘、可迭代的过程。
