# 产品思考

目前主要关注游戏产品数据和部分新兴产品，思考：为谁做，为什么现在做，做到什么程度算有效。

这里会放一些对产品问题的拆解。可能是一次指标变化，也可能是一个用户场景，或者是某个功能背后的取舍。

## 文章目录

- [A/B 实验方法论总览](ab_testing/index.md)

## 我比较关心

- 用户到底在什么场景里遇到了问题。
- 需求是真实存在，还是只是被包装出来的表达。
- 指标变化背后，哪些因素真的值得追。
- 功能优先级、增长路径和长期价值之间怎么取舍。

## 注意点

- 先把问题说清楚，再讨论方案。
- 数据可以帮助判断，但不能替代判断。
- 如果一个方案看起来合理，也要想清楚它该如何被验证。
